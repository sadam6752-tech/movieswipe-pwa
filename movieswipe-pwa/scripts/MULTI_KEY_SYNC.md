# Синхронизация с несколькими API ключами

## Как это работает

Скрипт `poiskkino-sync.cjs` теперь **автоматически определяет смену API ключа** и сбрасывает счетчик запросов, позволяя продолжить загрузку.

### Что отслеживается:
- ✅ **Прогресс загрузки** (какие фильмы уже загружены) - сохраняется
- ✅ **Хеш API ключа** - для определения смены ключа
- ✅ **Счетчик запросов** - сбрасывается при смене ключа
- ✅ **Курсор пагинации** - продолжает с того же места

## Использование

### Шаг 1: Запуск с первым ключом
```bash
cd movieswipe-pwa/scripts
node poiskkino-sync.cjs YOUR_API_KEY_1
```

Скрипт загрузит ~80,000 фильмов (200 запросов × 250 фильмов × рейтинг > 5.0)

### Шаг 2: Когда первый ключ исчерпается
Увидишь:
```
⚠️  Достигнут дневной лимит запросов (200) для текущего API ключа
💡 Вы можете продолжить с другим API ключом:
   node poiskkino-sync.cjs YOUR_OTHER_API_KEY
```

### Шаг 3: Продолжение со вторым ключом
```bash
node poiskkino-sync.cjs YOUR_API_KEY_2
```

Скрипт автоматически:
1. 🔑 Обнаружит новый API ключ
2. 🔄 Сбросит счетчик запросов (0/200)
3. ✅ Продолжит загрузку с того же места
4. 📊 Загрузит еще ~80,000 фильмов

### Результат
**Итого: ~160,000 фильмов за один день!** 🎉

## Технические детали

### Файл прогресса (.sync-progress.json)
```json
{
  "lastCursor": "eyJpZCI6MTIzNDU2fQ==",
  "totalMovies": 80000,
  "requestsToday": 200,
  "lastRequestDate": "2024-03-08",
  "apiKeyHash": "a1b2c3d4",  ← Хеш текущего ключа
  "yearRange": {
    "start": 2019,
    "end": 2024
  },
  "completed": false
}
```

### Что происходит при смене ключа:
1. Скрипт вычисляет MD5 хеш нового ключа
2. Сравнивает с сохраненным хешем
3. Если хеши разные:
   - Сбрасывает `requestsToday` → 0
   - Сбрасывает `lastRequestDate` → null
   - Обновляет `apiKeyHash` → новый хеш
   - **НЕ трогает** `lastCursor` и `totalMovies`

### Безопасность
- API ключ **не сохраняется** в файле
- Сохраняется только MD5 хеш (первые 8 символов)
- Невозможно восстановить ключ из хеша

## Сколько ключей можно использовать?

**Неограниченно!** Можешь использовать 3, 4, 5 ключей подряд:

```bash
node poiskkino-sync.cjs KEY_1  # 80k фильмов
node poiskkino-sync.cjs KEY_2  # +80k = 160k
node poiskkino-sync.cjs KEY_3  # +80k = 240k
node poiskkino-sync.cjs KEY_4  # +80k = 320k
```

## Часто задаваемые вопросы

### Q: Можно ли использовать один ключ несколько раз?
**A:** Да, но нужно подождать до следующего дня. Лимит 200 запросов/день на ключ.

### Q: Что если прервать загрузку?
**A:** Просто запусти скрипт снова с тем же или другим ключом. Прогресс сохранен.

### Q: Можно ли начать заново?
**A:** Да, удали файлы:
```bash
rm .sync-progress.json
rm ../public/data/movies-poiskkino.json
```

### Q: Как узнать сколько фильмов загружено?
**A:** Посмотри в `.sync-progress.json` поле `totalMovies` или:
```bash
node -e "console.log(require('../public/data/movies-poiskkino.json').length)"
```

## Поддержка

Если возникли проблемы:
1. Проверь API ключ (должен быть валидным)
2. Проверь интернет соединение
3. Посмотри логи в консоли
4. Проверь файл `.sync-progress.json`
