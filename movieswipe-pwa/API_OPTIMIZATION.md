# Оптимизация использования TMDB API

## Проблема

Изначальная реализация делала слишком много запросов:
- 50 запросов для получения списков (50 страниц × 20 фильмов = 1000 фильмов)
- 1000 запросов для получения деталей каждого фильма
- **Итого: ~1050 запросов** для загрузки 1000 фильмов

Это неэффективно и быстро исчерпывает лимиты бесплатного API.

## Решение

Используем гибридный подход:
1. **Базовая загрузка**: ~50 запросов для получения 1000 фильмов с базовой информацией
2. **Ленивая загрузка деталей**: 1 дополнительный запрос при открытии страницы фильма

### Базовая загрузка (50 запросов)
- Загружаются все фильмы с базовой информацией
- Название, описание, жанры, рейтинг, постеры
- Быстро (~15-20 секунд)

### Ленивая загрузка деталей (1 запрос на фильм)
- Загружается только при открытии страницы с описанием фильма
- Режиссер, актеры, точная длительность
- Детали кэшируются в IndexedDB
- Повторная загрузка не требуется

## Что доступно в базовом запросе?

Каждый фильм в списке популярных содержит:

### ✅ Доступно сразу (базовая загрузка)
- `id` - уникальный ID фильма
- `title` - название на русском
- `original_title` - оригинальное название
- `release_date` - дата выхода (год)
- `overview` - описание на русском
- `genre_ids` - массив ID жанров
- `vote_average` - рейтинг (0-10)
- `poster_path` - путь к постеру
- `backdrop_path` - путь к фоновому изображению

### 🔄 Загружается по требованию (при открытии фильма)
- `runtime` - точная длительность фильма
- `credits.crew` - режиссеры
- `credits.cast` - актеры (топ-5)

Детали загружаются один раз и кэшируются в IndexedDB.

## Компромиссы

### Что мы получаем?
- **Быстрая начальная загрузка** (~15-20 секунд для 1000 фильмов)
- **Минимальное использование API** (50 запросов для базы)
- **Полная информация по требованию** (1 запрос при открытии фильма)
- **Кэширование деталей** - повторная загрузка не требуется
- **Все ключевые данные** для выбора фильма сразу доступны

### Как это работает?
1. Пользователь загружает базу (50 запросов, 1000 фильмов)
2. Просматривает фильмы по названию, описанию, жанрам, рейтингу
3. Открывает интересующий фильм
4. Автоматически загружаются детали (режиссер, актеры, длительность)
5. Детали сохраняются в IndexedDB
6. При повторном открытии детали берутся из кэша

## Лимиты TMDB API

### Бесплатный план
- 40 запросов в 10 секунд
- Неограниченное количество запросов в день

### Наше использование
- 50 запросов для загрузки 1000 фильмов
- 250ms задержка между запросами (соблюдение rate limit)
- Время загрузки: ~15-20 секунд

### Сколько раз можно обновлять базу?
Теоретически неограниченно, но рекомендуется:
- 1-2 раза в день для обновления популярных фильмов
- Данные кэшируются локально в IndexedDB
- Работают offline после загрузки

## Альтернативные подходы

### Вариант 1: Только базовая информация (не реализован)
- Загружать базовую информацию для всех фильмов (~50 запросов)
- Не загружать детали вообще
- **Плюсы**: минимум запросов, быстро
- **Минусы**: нет информации о режиссерах и актерах

### Вариант 2: Гибридный подход (реализован) ✅
- Загружать базовую информацию для всех фильмов (~50 запросов)
- Загружать детали только для просматриваемых фильмов (по требованию)
- **Плюсы**: быстрая начальная загрузка + полная информация по требованию
- **Минусы**: дополнительный запрос при первом открытии фильма

### Вариант 3: Полная загрузка (не реализован)
- Загружать все детали сразу (~1050 запросов)
- **Плюсы**: вся информация сразу доступна
- **Минусы**: очень долго, много запросов

## Рекомендации

Текущий гибридный подход оптимален для большинства пользователей:
1. ✅ Быстрая загрузка базы (50 запросов)
2. ✅ Все необходимые данные для выбора фильма
3. ✅ Полная информация загружается по требованию
4. ✅ Детали кэшируются для offline доступа
5. ✅ Возможность частого обновления базы

### Типичный сценарий использования:
1. Пользователь загружает базу из 1000 фильмов (50 запросов, ~20 секунд)
2. Просматривает каталог, использует поиск и фильтры
3. Открывает 10-20 интересующих фильмов (10-20 дополнительных запросов)
4. **Итого**: 60-70 запросов для полноценного использования
5. При повторном использовании детали берутся из кэша (0 запросов)

## Статистика

### Базовая загрузка
- Запросов: ~50
- Время: ~15-20 секунд
- Данные: название, описание, жанры, рейтинг, постеры

### Загрузка деталей (по требованию)
- Запросов: 1 на фильм (только при первом открытии)
- Время: ~1 секунда на фильм
- Данные: режиссер, актеры, точная длительность
- Кэширование: детали сохраняются в IndexedDB

### Типичное использование
- Начальная загрузка: 50 запросов
- Просмотр 20 фильмов: +20 запросов
- **Итого**: ~70 запросов для полноценного использования
- Повторное использование: 0 запросов (все из кэша)

- ✅ Сохранить все ключевые данные для выбора фильма
- ✅ Предоставить полную информацию по требованию

- ✅ Экономить лимиты API
